1 # Conversation system # {#conversation_system}
2 [TOC]
3 
4 The Say and Portrait commands are a powerful way to build character dialogue, but it can be tedious to add large amounts of dialogue this way.
5 
6 The Conversation system provides a simplified format that allows you to quickly control:
7 
8 - Which character is speaking
9 - Which portrait image to use (either on the Say Dialog or on the Stage portrait)
10 - Which stage position to move to the character to (if using the Stage)
11 - When to hide a character
12 
13 To create a conversation, add the Conversation command (Narrative > Conversation) to a Block and enter the conversation text.
14 
15 # Example # {#example}
16 
17 ```text
18 john bored left: Oh, so that's how you use the Conversation command.
19 sherlock eyeroll right: Yes, well done John.
20 You catch on quickly don't you?
21 hide john "offscreen left": I sure do.
22 
23 -- This is a comment, it doesn't appear in the conversation
24 
25 john angry middle: Wait, what!
26 left: There's no need to be rude Sherlock!
27 bored: Not like that would stop you.
28 
29 sherlock excited: AHA! So that's how you do a conversation from Lua!
30 Fascinating.
31 john <<<: Yes, riveting.
32 
33 john hide:
34 sherlock hide:
35 ```

37 # Format # {#format}
38 
39 The format for conversation text is:
40 ```text
41 [character] [portrait] [position] [hide] [<<< | >>>] [clear | noclear] [wait | nowait] [fade | nofade]: [Dialogue text]
42 ```
43 
44 - character: The gameobject name or Name Text value of the speaking character.
45 - portrait: The name of a sprite in the character's Portraits list.
46 - position: The name of a position object in the Stage (e.g. Left, Middle, Right, Offscreen Left, Offscreen Right)
47 - hide: Hides the character
48 - <<<: Portrait face left
49 - >>>: Portrait face right
50 - clear | noclear: override the ClearPreviousLine default with true on clear or with false with noclear
51 - wait | nowait: override the WaitForInput default with true on wait or with false with nowait
52 - fade | nofade: override the FadeDone default with true on fade or with false with nofade. This is rarely required as it only really makes a difference on the last say of the conversation anyway.
53 
54 Parameters go on the left of the colon and the dialogue text goes on the right. You can omit any parameter and specify them in any order. Parameters are separated by spaces. If you need to use a name which contains spaces, wrap it in quotation marks e.g. "John Watson". Parameters are case insensitive. Blank lines and comment lines starting with -- are ignored. A line of dialogue text on its own will be spoken by the most recent character. You can omit dialogue text, but remember you still need to add the : character at the end of the line.

56 # String substitution # {#string_substitution}
57 
58 You can use the normal string subsitution syntax {$VarName} anywhere in the conversation text. For example if you have a string Flowchart variable called PlayerName you can embed this in a conversation like this:
59 
60 ```text
61 john: Hi there {$PlayerName}.
62 ```
63 
64 # Localization # {#localization}
65 
66 You can use the string substitution feature above with a @ref lua_string_table "Lua string table" to localize the conversation for multiple languages.
67 
68 The Conversation system does not work with the Localization component in %Fungus because the syntax makes it difficult to localize that way.
