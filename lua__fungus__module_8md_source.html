1 # %Fungus Lua Module # {#lua_fungus_module}
2 [TOC]
3 
4 The %Fungus Lua module provides handy functions for working with Lua, Unity and %Fungus. It is located in FungusLua/Resources/Lua/Fungus.txt and is loaded automatically when you use the Lua Script component.
5 
6 In this page we cover some of the more generic functionality in the module, other major features are described elsewhere in the documentation.

8 # Inspecting Lua objects # {#inspecting_lua_objects}
9 
10 You can use Lua's built in print() function to get a basic description of any object printed to the console. When you want to get a more detailed description of an object, use inspect().
11 
12 ```lua
13 -- Prints a short description of object v
14 print(v)
15 
16 -- Prints a summary of object v in a human readable format.
17 inspect(v)
18 ```

20 # Running Unity coroutines # {#run_unity_coroutines}
21 
22 When you bind to a C# component using Lua Bindings, you can access any public method in the class. If a method returns IEnumerator then that method can be executed as a [Unity coroutine], which is a powerful way to run asynchronous code.
23 
24 The runwait() function allows you to call a C# coroutine method from Lua which may take multiple frames to finish its work, and then carry on with the rest of the Lua code once that C# method has finished executing. This is how the say() function works for example.
25 
26 This is the list of available functions for waiting and working with coroutines.
27 
28 ```lua
29 -- Waits for a number of seconds, then continue execution of Lua script
30 wait(duration)
31 
32 -- Waits until the Lua function provided returns true, or the timeout expires.
33 -- Returns true if the function succeeded, or false if the timeout expired
34 waitfor(fn, timeoutduration)
35 
36 -- Run a C# coroutine and continue execution of Lua script
37 run(co)
38 
39 -- Run a C# coroutine, wait until it completes, then continue execution of Lua script
40 runwait(co)
41 ```

43 # Globals vs Table mode # {#globals_vs_tables}
44 
45 The %Fungus module can be used in three modes, controlled by the %Fungus Module option in the LuaUtils component.
46 
47 1. Use Global Variables: all module functions are mapped to global functions. This allows for convenient access, but it runs the risk that you might accidentally declare a variable with the same name as a %Fungus module function.
48 2. Use %Fungus Variable: all module functions are accessed through a global table called 'fungus'. This gives a degree of namespace safety at the cost of more typing.
49 3. No %Fungus Module: the %Fungus module will not be registered. Used if you don't want to use the %Fungus module.
50 
51 Options 1 and 2 are functionaly equivalent, it's just a matter of personal preference which you want to use.
52 
53 ```lua
54 -- sub is a function in the %Fungus module, mapped to a global variable
55 
56 -- Use Global Variables
57 sub('a string')
58 
59 -- Use Fungus Variable
60 fungus.sub('a string')
61 ```
